# 제1부: 시스템 설치

---

## 1장. 설치 전 준비사항

### 1.1 시스템이란?

이 시스템은 **법인카드 사용을 실시간으로 모니터링하고 관리**하는 솔루션입니다.

| 구성 요소 | 설명 | 사용 대상 |
|-----------|------|-----------|
| **관리자 대시보드** | 웹 브라우저에서 사용하는 관리 화면 | 관리자, 재무팀, 팀장 |
| **모바일 앱** | 직원이 영수증을 촬영·제출하는 앱 | 전 직원 |
| **백엔드 API 서버** | 모든 데이터를 처리하는 서버 | (자동 실행) |
| **데이터베이스/캐시** | 데이터 저장소 | (자동 실행) |

### 1.2 내 컴퓨터 사양 확인

다음 최소 사양을 충족해야 합니다:

| 항목 | 최소 사양 | 권장 사양 |
|------|-----------|-----------|
| 운영체제 | Windows 10 / macOS 12 | Windows 11 / macOS 14 |
| RAM | 8GB | 16GB |
| 저장 공간 | 10GB 여유 공간 | 20GB 여유 공간 |
| 인터넷 | 필수 (설치 시) | 상시 연결 |

### 1.3 확인 방법 (Windows)

1. 키보드에서 `Win + I` 를 눌러 **설정** 열기
2. **시스템** → **정보** 클릭
3. "설치된 RAM"과 "시스템 종류"를 확인합니다

## 2장. 필수 프로그램 설치

시스템을 실행하려면 아래 3가지 프로그램을 먼저 설치해야 합니다.

### 2.1 Node.js 설치 (JavaScript 실행 환경)

Node.js는 서버와 대시보드를 실행하는 데 필요합니다.

**설치 단계:**

1. 웹 브라우저에서 Node.js 공식 사이트에 접속합니다
2. **"LTS"** (안정 버전) 다운로드 버튼을 클릭합니다
3. 다운로드된 설치 파일(`.msi`)을 더블 클릭합니다
4. 설치 마법사가 나타나면:
   - `Next` 클릭
   - "I accept the terms" 체크 → `Next` 클릭
   - 설치 경로 변경 없이 `Next` 클릭
   - **"Automatically install the necessary tools"** 체크 → `Next` 클릭
   - `Install` 클릭
5. 설치 완료 후 `Finish` 클릭

**설치 확인:**

1. `Win + R` → `cmd` 입력 → Enter (명령 프롬프트 열기)
2. 다음을 입력하고 Enter:
   ```
   node --version
   ```
3. `v20.x.x` 이상의 버전이 표시되면 성공입니다

   ```
   ✅ 정상: v22.16.0
   ❌ 실패: 'node'은(는) 내부 또는 외부 명령...이 아닙니다
   ```

> **오류 시**: 컴퓨터를 재시작한 후 다시 시도하세요. 재시작 후에도 안 되면 [17장. 설치 관련 오류](#17장-설치-관련-오류)를 참고하세요.

---

### 2.2 Docker Desktop 설치 (데이터베이스 실행 환경)

Docker는 데이터베이스(PostgreSQL), 캐시(Redis), 파일 저장소(S3)를 쉽게 실행합니다.

**설치 단계:**

1. Docker Desktop 공식 사이트에 접속합니다
2. **"Download for Windows"** 버튼을 클릭합니다
3. 다운로드된 `Docker Desktop Installer.exe`를 실행합니다
4. 설치 옵션:
   - **"Use WSL 2 instead of Hyper-V"** 체크 (권장)
   - `Ok` 클릭
5. 설치 완료 후 **컴퓨터를 재시작**합니다
6. 재시작 후 Docker Desktop이 자동으로 시작됩니다
   - 작업 표시줄 오른쪽 하단에 🐳 고래 아이콘이 나타납니다
   - 아이콘이 움직이다가 멈추면 준비 완료입니다

**설치 확인:**

1. 명령 프롬프트에서:
   ```
   docker --version
   ```
2. `Docker version 27.x.x` 등이 표시되면 성공입니다

> **주의**: Docker Desktop은 **항상 실행 중이어야** 합니다. 컴퓨터 시작 시 자동 실행을 권장합니다.
> Docker Desktop → Settings(⚙️) → General → **"Start Docker Desktop when you sign in to your computer"** 체크

---

### 2.3 Git 설치 (소스 코드 관리)

Git은 프로젝트 소스 코드를 다운로드하고 관리하는 데 필요합니다.

**설치 단계:**

1. Git 공식 사이트에 접속합니다
2. **"Download for Windows"** 클릭
3. 설치 파일 실행 후, 모든 옵션을 **기본값 그대로** `Next` 클릭
4. `Install` → `Finish`

**설치 확인:**

```
git --version
```

`git version 2.x.x` 이 표시되면 성공입니다.

## 3장. 시스템 설치 및 실행

### 3.1 프로젝트 폴더 열기

프로젝트 소스 코드가 있는 폴더를 준비합니다.

- 소스 코드를 전달받았다면, 해당 폴더로 이동합니다
- Git 저장소에서 받는 경우, 명령 프롬프트에서:
  ```
  cd 원하는경로
  git clone [저장소 주소]
  ```

이후 모든 명령어는 **프로젝트 루트 폴더**에서 실행합니다.

---

### 3.2 의존성 설치 (npm install)

프로젝트에 필요한 라이브러리를 자동으로 다운로드합니다.

**실행 방법:**

1. 프로젝트 폴더에서 명령 프롬프트를 엽니다
   - 폴더 탐색기에서 주소 표시줄에 `cmd` 입력 → Enter
2. 다음 명령어를 입력합니다:
   ```
   npm install
   ```
3. 설치가 진행되며, 완료까지 1~3분 소요됩니다

**정상 완료 화면:**
```
added 1586 packages in 11s
```

> **오류 시**: [17장. 설치 관련 오류](#17장-설치-관련-오류) 참고

---

### 3.3 인프라 시작 (Docker)

데이터베이스, 캐시, 파일 저장소를 Docker로 실행합니다.

**실행 방법:**

1. Docker Desktop이 실행 중인지 확인합니다 (작업 표시줄 🐳 아이콘)
2. 명령 프롬프트에서:
   ```
   docker compose up -d
   ```
3. 처음 실행 시 이미지 다운로드로 3~5분 소요됩니다

**정상 완료 화면:**
```
✔ Container corporate-card-db    Started
✔ Container corporate-card-redis  Started
✔ Container corporate-card-s3     Started
```

**상태 확인:**
```
docker compose ps
```

세 개의 서비스 모두 **"Up ... (healthy)"** 상태여야 합니다:

```
NAME                   STATUS
corporate-card-db      Up 5 minutes (healthy)   ← PostgreSQL
corporate-card-redis   Up 5 minutes (healthy)   ← Redis
corporate-card-s3      Up 5 minutes (healthy)   ← S3 파일 저장소
```

> **"unhealthy" 또는 "Exited" 상태인 경우**: [18장. 서버 실행 오류](#18장-서버-실행-오류) 참고

### 3.4 환경변수 설정

서버가 데이터베이스 등에 접속하기 위한 설정 파일을 만듭니다.

**이미 `.env.development` 파일이 있는 경우** (대부분의 배포 환경):
- 추가 작업 없이 다음 단계로 진행합니다

**새로 만들어야 하는 경우:**

1. `packages/api/` 폴더로 이동합니다
2. `.env.example` 파일을 복사하여 `.env.development`로 이름을 바꿉니다
   ```
   copy packages\api\.env.example packages\api\.env.development
   ```
3. `.env.development` 파일을 메모장으로 열어 다음 값을 수정합니다:

| 항목 | 수정 전 (기본값) | 수정 후 |
|------|------------------|---------|
| `DB_PASSWORD` | `your_password` | `password` |
| `JWT_SECRET` | `your_jwt_secret_key` | `dev-jwt-secret-key-for-local-testing-2024` |
| `JWT_REFRESH_SECRET` | `your_refresh_secret_key` | `dev-jwt-refresh-secret-key-for-local-testing-2024` |
| `ENCRYPTION_KEY` | `your_encryption_key_min_32_chars` | `dev-encryption-key-32chars-min!!` |
| `AWS_ACCESS_KEY_ID` | `your_access_key` | `test` |
| `AWS_SECRET_ACCESS_KEY` | `your_secret_key` | `test` |

4. 파일을 저장합니다

> **중요**: `DB_PASSWORD`는 Docker에서 설정한 비밀번호(`password`)와 반드시 일치해야 합니다.

## 4장. 초기 설정 마법사

시스템을 처음 설치하면, 관리자가 외부 서비스 연결을 설정하는 **초기 설정 마법사**가 자동으로 나타납니다.

### 4.1 설정 마법사 진입

1. 브라우저에서 `http://localhost:5173` 접속
2. 관리자 계정으로 로그인:
   - **사번**: `ADMIN001`
   - **비밀번호**: `password123`
3. 초기 설정이 필요하면 자동으로 설정 마법사 화면으로 이동합니다

화면 상단에 **5단계 진행 표시줄**이 나타납니다:
```
[1.Database] → [2.Redis] → [3.S3] → [4.OCR] → [5.LLM]
```

---

### 4.2 Step 1: Database (PostgreSQL) 연결

데이터베이스 서버 연결 정보를 입력합니다.

| 입력 항목 | 입력 값 | 설명 |
|-----------|---------|------|
| Host | `localhost` | 같은 컴퓨터에서 실행 중이므로 localhost |
| Port | `5432` | PostgreSQL 기본 포트 |
| Username | `postgres` | 데이터베이스 사용자 이름 |
| Password | `password` | docker-compose.yml에 설정된 비밀번호 |
| Database | `corporate_card` | 데이터베이스 이름 |

**설정 방법:**

1. 각 항목에 위 값을 입력합니다
2. **[연결 테스트]** 버튼을 클릭합니다
3. "연결 성공" 메시지가 나타나면 **[저장 후 다음]** 클릭

**오류가 나는 경우:**

| 오류 메시지 | 원인 | 해결 방법 |
|-------------|------|-----------|
| "연결 거부됨" | Docker가 실행 중이 아님 | Docker Desktop 실행 확인, `docker compose ps`로 상태 확인 |
| "인증 실패" | 비밀번호 불일치 | `password`를 정확히 입력했는지 확인 |
| "데이터베이스 없음" | DB가 생성 안 됨 | `docker compose down -v && docker compose up -d` 실행 |

---

### 4.3 Step 2: Redis 연결

캐시 서버(세션 관리, 토큰 저장) 연결 정보를 입력합니다.

| 입력 항목 | 입력 값 | 설명 |
|-----------|---------|------|
| Host | `localhost` | 같은 컴퓨터 |
| Port | `6379` | Redis 기본 포트 |
| Password | (비워 둠) | 개발 환경에서는 비밀번호 없음 |

**설정 방법:**

1. Host와 Port를 입력합니다 (Password는 비워 둡니다)
2. **[연결 테스트]** 클릭 → "연결 성공" 확인
3. **[저장 후 다음]** 클릭

---

### 4.4 Step 3: S3 파일 저장소

영수증 이미지를 저장할 파일 저장소를 설정합니다. 로컬 개발 환경에서는 LocalStack(S3 시뮬레이터)을 사용합니다.

| 입력 항목 | 입력 값 | 설명 |
|-----------|---------|------|
| Region | `ap-northeast-2` | 서울 리전 |
| Endpoint | `http://localhost:4566` | LocalStack 주소 |
| Access Key ID | `test` | 개발용 키 |
| Secret Access Key | `test` | 개발용 키 |
| Bucket | `corporate-card-receipts` | 버킷 이름 |

**설정 방법:**

1. 각 항목에 위 값을 입력합니다
2. **[연결 테스트]** 클릭 → "연결 성공" 확인
3. **[저장 후 다음]** 클릭

> **운영 환경에서는** AWS의 실제 S3 정보를 입력합니다. 이 경우 Endpoint는 비워 둡니다.

---

### 4.5 Step 4: OCR (영수증 인식)

영수증 사진에서 텍스트를 자동 추출하는 OCR 서비스를 연결합니다.

| 입력 항목 | 입력 값 | 설명 |
|-----------|---------|------|
| API URL | (비워 둠) | NAVER CLOVA OCR API 주소 |
| Secret Key | (비워 둠) | NAVER CLOVA 인증키 |

**OCR 서비스 없이 진행하는 경우:**

- 아직 NAVER CLOVA OCR 계정이 없다면, 비워 두고 **[저장 후 다음]**을 클릭합니다
- OCR 기능만 비활성화되며, 나머지 기능은 정상 작동합니다
- 나중에 **설정 → 외부 서비스** 탭에서 언제든 설정할 수 있습니다

**NAVER CLOVA OCR를 사용하는 경우:**

1. NAVER Cloud Platform에서 CLOVA OCR 서비스를 신청합니다
2. 발급받은 API URL과 Secret Key를 입력합니다
3. **[연결 테스트]** 클릭 → "연결 성공" 확인
4. **[저장 후 다음]** 클릭

---

### 4.6 Step 5: LLM (AI 보정)

OCR 결과를 AI가 자동으로 보정해주는 LLM 서비스를 연결합니다.

| 입력 항목 | 설명 |
|-----------|------|
| 프로바이더 | AI 서비스 제공자 선택 |
| API Key | 해당 서비스의 인증 키 |
| 모델 | 사용할 AI 모델 |
| Endpoint | (커스텀 프로바이더만 해당) |

**프로바이더별 설정:**

| 프로바이더 | 기본 모델 | API Key 형태 |
|-----------|-----------|-------------|
| Anthropic (Claude) | claude-sonnet-4-20250514 | `sk-ant-...` |
| OpenAI (GPT) | gpt-4o | `sk-...` |
| Google (Gemini) | gemini-2.0-flash | `AIza...` |
| Custom | (직접 입력) | (서비스별 상이) |

**설정 방법:**

1. 드롭다운에서 프로바이더를 선택합니다
2. API Key를 입력합니다
3. 모델이 자동으로 채워집니다 (변경 가능)
4. **[연결 테스트]** 클릭 → "연결 성공" 확인
5. **[완료]** 클릭

**LLM 없이 진행하는 경우:**

- API Key가 없다면, 비워 두고 **[완료]**를 클릭합니다
- AI 보정 기능만 비활성화되고, OCR 원본 결과가 그대로 사용됩니다

## 6장. 대시보드 (메인 화면)

로그인 후 가장 먼저 나타나는 화면입니다. 법인카드 사용 현황을 한눈에 볼 수 있습니다.

```
┌─────────────────────────────────────────────────┐
│  🔔 알림  │  사용자 이름 (부서)  │  로그아웃     │  ← 상단 헤더

│      │                                         │

├──────┘                                         │
└─────────────────────────────────────────────────┘
```

### 6.2 4개 지표 카드

화면 상단에 4개의 핵심 지표가 표시됩니다:

| 카드 | 표시 내용 | 예시 |
|------|-----------|------|
| **금일 사용액** | 오늘 사용된 법인카드 총액 + 전일 대비 증감 | ₩1,234,000 (+15%) |
| **월 사용액** | 이번 달 누적 사용액 + 예산 대비 비율 | ₩15,000,000 (60%) |
| **승인률** | 승인된 거래 / 전체 거래 비율 | 95.2% |
| **위반 건수** | 정책 위반으로 거절된 거래 수 | 3건 |

### 6.3 실시간 거래 피드

최근 10건의 거래가 시간순으로 표시됩니다. **30초마다 자동 갱신**됩니다.

각 거래 항목에 표시되는 정보:
- 시간 (예: 14:30)
- 직원 이름, 부서
- 가맹점 이름
- 결제 금액
- 상태: 🟢 승인 / 🔴 거절 / 🟡 주의 / 🔵 대기

### 6.4 부서별 사용 현황

각 부서의 예산 사용률이 막대 그래프로 표시됩니다.

- **파란색 막대**: 정상 사용 (80% 미만)
- **빨간색 막대**: 초과 위험 (80% 이상)

예시:
```
경영지원팀  ████████░░  80%  ₩800만 / ₩1000만
개발팀      █████░░░░░  50%  ₩500만 / ₩1000만
영업팀      ██████████  100% ₩1200만 / ₩1200만  ← 빨간색!
```

### 6.5 사용 추이 차트

최근 30일간의 일별 법인카드 사용액을 꺾은선 그래프로 보여줍니다.

- X축: 날짜
- Y축: 금액 (만원 단위)
- 마우스를 올리면 해당 날짜의 정확한 금액이 표시됩니다

### 6.6 위반 요약

정책 위반 유형별 건수가 정리됩니다:

| 위반 유형 | 설명 |
|-----------|------|
| 위치 불일치 | GPS 위치와 영수증 주소가 다름 |
| 업종 제한 | 사용 불가 업종에서 결제 |
| 지역 위반 | 허용되지 않은 지역에서 결제 |
| 한도 초과 | 건별/일별/월별 한도 초과 |

### 6.7 알림 확인

상단 헤더의 **🔔 알림** 아이콘을 클릭하면:

- 읽지 않은 알림 수가 빨간 숫자로 표시됩니다
- 알림을 클릭하면 해당 거래 상세 페이지로 이동합니다

### 6.8 좌측 메뉴 접기/펼치기

화면 왼쪽 상단의 **≡ 햄버거 아이콘**을 클릭하면 메뉴를 접거나 펼 수 있습니다.

## 7장. 거래 내역 관리

좌측 메뉴에서 **거래 내역**을 클릭합니다.

### 7.1 거래 목록 화면

모든 법인카드 거래가 표 형태로 표시됩니다.

**표시되는 정보:**

| 열 이름 | 설명 | 정렬 가능 |
|---------|------|-----------|
| 거래일시 | 카드 결제 시간 | ✅ |
| 직원명 | 결제한 직원 이름 | |
| 부서 | 소속 부서 | |
| 가맹점 | 결제 장소 | |
| 업종 | 가맹점 업종 (식음료, 교통 등) | |
| 금액 | 결제 금액 | ✅ |
| 상태 | 승인/거절/주의/대기 | |
| 액션 | 상세 보기 버튼 | |

### 7.2 거래 검색 및 필터

화면 상단의 검색 영역에서 원하는 조건으로 거래를 찾을 수 있습니다:

| 필터 항목 | 사용 방법 | 예시 |
|-----------|-----------|------|
| **검색** | 직원명 또는 가맹점명 입력 | "김민수" 또는 "스타벅스" |
| **상태** | 드롭다운에서 선택 | 승인 / 거절 / 주의 / 대기 |
| **부서** | 드롭다운에서 선택 | 경영지원팀, 개발팀 등 |
| **기간** | 시작일 ~ 종료일 선택 | 2026-03-01 ~ 2026-03-08 |
| **금액 범위** | 최소 ~ 최대 입력 | 10,000 ~ 500,000 |

**검색 예시: "이번 달 거절된 거래만 보기"**

1. 기간: 이번 달 1일 ~ 오늘
2. 상태: **거절** 선택
3. **[검색]** 클릭

**필터 초기화:**

- **[초기화]** 버튼을 클릭하면 모든 필터가 리셋됩니다

### 7.3 거래 상세 보기

거래 목록에서 **[상세]** 버튼을 클릭하면 모달(팝업 창)이 열립니다.

**표시 내용:**

1. **직원 정보**: 이름, 부서
2. **거래 정보**: 가맹점, 업종, 금액, 일시, 현재 상태
3. **위치 정보**: 결제 위치 주소, 위도/경도
4. **검증 결과**: 4가지 자동 검증 결과
   - ✅ 업종: 허용된 업종인지
   - ✅ 위치: GPS 위치와 영수증 주소 일치 여부 (거리 표시)
   - ✅ 한도: 한도 이내인지
   - ✅ 지역: 허용된 지역인지

### 7.4 거래 승인/거절 처리

**대기(pending)** 또는 **주의(flagged)** 상태인 거래에 대해 처리할 수 있습니다.

> ⚠️ 이 기능은 **관리자, 재무팀, 팀장** 역할만 사용할 수 있습니다.

**승인하기:**
1. 거래 상세 모달에서 **[승인]** 버튼(초록색)을 클릭합니다
2. 확인 메시지가 나타납니다
3. **[확인]**을 클릭하면 거래가 "승인" 상태로 변경됩니다

**거절하기:**
1. 거래 상세 모달에서 **[거절]** 버튼(빨간색)을 클릭합니다
2. 거절 사유를 입력합니다 (필수)
3. **[확인]**을 클릭하면 거래가 "거절" 상태로 변경됩니다

**플래그 지정하기:**
1. **[플래그 지정]** 버튼(노란색)을 클릭합니다
2. 해당 거래가 "주의" 상태로 표시됩니다
3. 추후 추가 확인이 필요한 건으로 분류됩니다

## 8장. 직원 관리

좌측 메뉴에서 **직원 관리**를 클릭합니다.

> ⚠️ 이 메뉴는 **관리자, 재무팀, 팀장** 역할만 접근 가능합니다.

### 8.1 직원 목록 화면

등록된 모든 직원이 표 형태로 표시됩니다.

**표시 정보:**

| 열 이름 | 설명 |
|---------|------|
| 사번 | 직원 고유 번호 (예: EMP001) |
| 이름 | 직원 이름 |
| 부서 | 소속 부서 |
| 직급 | 사원, 대리, 과장, 차장, 부장 |
| 월 한도 | 법인카드 월 사용 한도 |
| 한도 사용률 | 현재 월 사용액 / 월 한도 (%) |
| 상태 | 🟢 활성 또는 ⚪ 비활성 |
| 액션 | 보기 / 편집 버튼 |

**한도 사용률 색상:**
- 파란색: 80% 미만 (정상)
- 빨간색: 80% 이상 (주의)

### 8.2 직원 검색

| 검색 항목 | 방법 |
|-----------|------|
| 이름 검색 | 검색창에 직원 이름 입력 |
| 부서 필터 | 드롭다운에서 부서 선택 |

### 8.3 새 직원 등록

> ⚠️ **관리자(ADMIN)** 역할만 가능합니다.

1. 화면 오른쪽 상단의 **[직원 등록]** 버튼을 클릭합니다
2. 입력 양식이 모달로 나타납니다:

| 항목 | 필수 여부 | 설명 | 입력 예시 |
|------|-----------|------|-----------|
| 사번 | ✅ 필수 | 직원 고유 번호 | `EMP010` |
| 이름 | ✅ 필수 | 직원 이름 | `김민수` |
| 이메일 | ✅ 필수 | 회사 이메일 | `minsu@company.com` |
| 전화번호 | 선택 | 연락처 | `010-1234-5678` |
| 부서 | ✅ 필수 | 소속 부서 선택 | `개발팀` |
| 직급 | 선택 | 직급 선택 | `대리` |
| 역할 | ✅ 필수 | 시스템 권한 | `employee` |
| 월 한도 | 선택 | 월 사용 한도 | `2,000,000` |
| 활성 상태 | 선택 | 계정 활성화 토글 | ON/OFF |

3. 모든 필수 항목을 입력한 후 **[등록]** 클릭

**역할(권한) 설명:**

| 역할 | 영문 | 권한 |
|------|------|------|
| 직원 | `employee` | 모바일 앱에서 영수증 제출만 가능 |
| 팀장 | `manager` | 소속 팀원의 거래 승인/거절 가능 |
| 재무 | `finance` | 전체 거래 관리, 정책 설정 가능 |
| 관리자 | `admin` | 모든 기능 사용 가능 |

### 8.4 직원 정보 수정

1. 직원 목록에서 해당 직원의 **[✏️ 편집]** 아이콘을 클릭합니다
2. 수정할 항목을 변경합니다
   - 사번은 수정 불가 (회색으로 표시)
3. **[수정]** 클릭

### 8.5 직원 상세 보기

직원 목록에서 **[👁 보기]** 아이콘을 클릭하면:

- 기본 정보: 사번, 이름, 이메일, 전화번호, 부서, 직급, 역할, 상태, 등록일
- **카드 한도 현황**:
  - 사용액 / 한도 (예: ₩1,500,000 / ₩2,000,000)
  - 사용률 막대 그래프
  - 잔여 한도 표시

## 9장. 정책 관리

좌측 메뉴에서 **정책 관리**를 클릭합니다.

> ⚠️ **관리자(ADMIN)** 역할만 정책 생성/수정/삭제가 가능합니다.

### 9.1 정책이란?

**정책**은 법인카드 사용 규칙의 모음입니다. 예를 들어:

- "유흥업소에서 결제하면 자동 차단"
- "건당 50만원 이상이면 승인 필요"
- "허용 지역 외에서 결제하면 알림"

하나의 정책은 여러 개의 **규칙**으로 구성됩니다.

### 9.2 정책 목록 화면

| 열 이름 | 설명 |
|---------|------|
| 정책명 | 정책의 이름 |
| 설명 | 정책에 대한 간단한 설명 |
| 규칙 수 | 포함된 규칙의 개수 |
| 적용 부서 | 이 정책이 적용되는 부서들 |
| 활성 | ON/OFF 토글 스위치 |
| 수정일 | 마지막으로 수정한 날짜 |
| 액션 | 보기 / 편집 / 삭제 |

### 9.3 새 정책 만들기

**예시: "유흥업소 결제 차단" 정책 만들기**

1. **[정책 생성]** 버튼 클릭
2. 기본 정보 입력:
   - **정책명**: `유흥업소 결제 차단`
   - **설명**: `유흥 및 도박 관련 업종에서의 결제를 차단합니다`
   - **적용 부서**: `전체` (또는 특정 부서 선택)

3. 규칙 추가:
   - **[+ 규칙 추가]** 버튼을 클릭합니다

   | 항목 | 선택/입력 |
   |------|-----------|
   | 유형 | `업종` |
   | 조건 | `equals` |
   | 값 | `유흥` |
   | 조치 | `차단 (deny)` |

4. 규칙을 더 추가하려면 **[+ 규칙 추가]**를 다시 클릭합니다
5. 모든 규칙 입력 후 **[등록]** 클릭

**규칙 유형 설명:**

| 유형 | 설명 | 예시 |
|------|------|------|
| 업종 (category) | 가맹점 업종 기준 | 유흥, 도박, 사행 |
| 지역 (region) | 결제 지역 기준 | 서울, 경기 |
| 시간 (time) | 결제 시간 기준 | 22:00 이후 |
| 금액 (amount) | 결제 금액 기준 | 500,000원 이상 |
| 가맹점 (merchant) | 특정 가맹점 기준 | 특정 가맹점명 |

**조치(Action) 설명:**

| 조치 | 색상 | 설명 |
|------|------|------|
| 차단 (deny) | 🔴 빨간색 | 결제가 자동 거절됩니다 |
| 주의 (flag) | 🟡 노란색 | 결제는 되지만 관리자에게 알림이 갑니다 |
| 허용 (allow) | 🟢 초록색 | 명시적으로 허용 (다른 규칙보다 우선) |

### 9.4 정책 활성화/비활성화

정책 목록의 **활성** 열에 있는 토글 스위치를 클릭합니다:
- **ON** (파란색): 정책이 적용 중
- **OFF** (회색): 정책이 일시 중지

> 비활성화된 정책은 삭제하지 않고도 잠시 중단할 수 있습니다.

### 9.5 정책 삭제

1. 해당 정책의 **[🗑 삭제]** 아이콘을 클릭합니다
2. "정말 삭제하시겠습니까?" 확인 대화상자가 나타납니다
3. **[확인]**을 클릭하면 삭제됩니다

> ⚠️ 삭제된 정책은 복구할 수 없습니다. 필요하다면 비활성화를 먼저 고려하세요.

## 10장. 통계 및 리포트

좌측 메뉴에서 **통계/리포트**를 클릭합니다.

### 10.1 필터 설정

화면 상단에서 조회 조건을 설정합니다:

| 필터 | 설명 | 기본값 |
|------|------|--------|
| 부서 | 특정 부서만 보기 | 전체 |
| 기간 | 시작일 ~ 종료일 | 최근 30일 |

### 10.2 요약 지표 (3개 카드)

| 카드 | 표시 내용 |
|------|-----------|
| **총 사용액** | 선택 기간 내 전체 사용 금액 + 전기 대비 증감률 |
| **총 거래 건수** | 선택 기간 내 총 결제 건수 |
| **승인률** | 승인 건수 / 전체 건수 (%) |

### 10.3 일별 사용 추이 (꺾은선 그래프)

- 선택 기간 동안 **매일 사용된 총액**이 선 그래프로 표시됩니다
- 마우스를 올리면 정확한 날짜와 금액이 보입니다

**활용 예시:**
- 월말에 사용이 급증하는 패턴 파악
- 특정 날짜에 비정상적으로 높은 사용액 감지

### 10.4 카테고리별 지출 분포 (파이 차트)

법인카드가 어떤 업종에 주로 사용되는지 비율로 보여줍니다.

예시:
```
🔵 식음료    35%
🟢 교통      25%
🟡 사무용품  20%
🟠 숙박      10%
🔴 기타      10%
```

### 10.5 부서별 사용 비교 (막대 그래프)

각 부서의 **실제 사용액**과 **예산 한도**를 나란히 비교합니다.

- 파란색 막대: 실제 사용액
- 회색 막대: 예산 한도
- 파란 막대가 회색 막대를 넘으면 예산 초과!

초기 설정 마법사에서 설정한 외부 서비스의 연결 상태를 확인하고 변경할 수 있습니다.

각 서비스 옆에 상태가 표시됩니다:

| 상태 | 색상 | 의미 |
|------|------|------|
| 연결됨 | 🟢 초록 | 정상 연결 중 |
| 미설정 | ⚪ 회색 | 아직 설정하지 않음 |
| 연결 실패 | 🔴 빨간 | 연결에 문제가 있음 |
| 확인 중 | 🟡 노란 | 연결 테스트 진행 중 |

**설정 가능한 서비스:**

1. **Database (PostgreSQL)**: Host, Port, Username, Password, DB명
2. **Redis**: Host, Port, Password
3. **S3 Storage**: Region, Endpoint, Access Key, Secret Key, Bucket
4. **OCR (CLOVA)**: API URL, Secret Key

각 서비스마다 **[연결 테스트]** 버튼으로 연결 상태를 확인할 수 있습니다.

### 11.4 LLM 설정 탭

AI 보정 서비스의 프로바이더를 선택하고 설정합니다.

| 설정 항목 | 설명 |
|-----------|------|
| 프로바이더 | Anthropic / OpenAI / Google / Custom |
| API Key | 해당 서비스의 인증 키 (비밀번호처럼 ***로 표시) |
| 모델 | 사용할 AI 모델 (프로바이더에 따라 자동 선택) |
| Endpoint | Custom 프로바이더인 경우만 입력 |

**프로바이더 변경 방법:**

1. 드롭다운에서 원하는 프로바이더를 선택합니다
2. API Key를 입력합니다
3. **[연결 테스트]** 클릭하여 동작 확인
4. **[저장]** 클릭

## 14장. 영수증 제출

법인카드로 결제 후, 영수증을 촬영하여 제출하는 과정입니다.

### 14.1 전체 흐름

```
[1. 영수증 촬영] → [2. 내용 확인] → [3. 검증 진행] → [4. 결과 확인]
```

### 14.2 Step 1: 영수증 촬영 (Upload 화면)

하단 탭의 **📷 영수증**을 터치하거나, 홈 화면의 **[영수증 업로드하기]** 버튼을 터치합니다.

**① 사진 촬영/선택**

| 버튼 | 기능 |
|------|------|
| 📷 카메라 | 카메라를 열어 영수증 사진을 직접 촬영 |
| 🖼 갤러리 | 이미 촬영한 사진을 갤러리에서 선택 |

- 사진을 선택하면 **미리보기**가 표시됩니다
- 다시 찍고 싶으면 **[다시 선택]** 을 터치합니다

> **촬영 팁:**
> - 영수증 전체가 프레임 안에 들어오게 촬영하세요
> - 글자가 선명하게 보이도록 밝은 곳에서 촬영하세요
> - 구겨지거나 접힌 부분이 없도록 펼쳐서 촬영하세요
> - 최대 파일 크기: 10MB

**② GPS 위치 확인**

사진을 선택하면 현재 위치가 자동으로 감지됩니다:

```
📍 서울시 강남구 테헤란로 123
   정확도: 좋음 (15m)
```

위치 정확도:
| 정확도 | 색상 | 의미 |
|--------|------|------|
| 좋음 | 🟢 초록 | 15m 이내 (GPS 수신 양호) |
| 보통 | 🟡 노란 | 15~50m (건물 실내 등) |
| 나쁨 | 🔴 빨간 | 50m 이상 (GPS 수신 불량) |

> 위치가 잘 안 잡히면 **[위치 재시도]** 버튼을 터치하세요.

**③ 추가 정보 (선택)**

| 항목 | 설명 |
|------|------|
| 메모 | 결제 목적 등을 자유롭게 입력 (예: "팀 회식") |
| 배달 주문 | 배달 주문인 경우 체크 (위치 불일치 방지) |
| 온라인 결제 | 온라인 결제인 경우 체크 |

**④ 업로드**

사진과 위치가 모두 준비되면 **[업로드]** 버튼이 활성화됩니다. 터치하여 제출합니다.

### 14.3 Step 2: 내용 확인 (Confirm 화면)

OCR(광학 문자 인식)이 영수증에서 자동으로 읽어낸 내용을 확인하고 수정합니다.

**AI가 자동 보정한 경우:**

```
🤖 AI가 다음 항목을 자동으로 수정했습니다:
  • 금액: 50,0OO → 50,000
  • 상호: 스타B벅스 → 스타벅스
```

**확인할 항목:**

| 항목 | 설명 | 수정 가능 |
|------|------|-----------|
| 상호명 | 가맹점 이름 | ✅ |
| 금액 | 결제 금액 | ✅ |
| 부가세 | 부가가치세 | ✅ |
| 업종 | 카테고리 선택 (5개 중 택 1) | ✅ |
| 주소 | 가맹점 주소 | ✅ |
| 거래 일시 | 결제 시간 | ✅ |

**업종 선택:**

다음 5가지 버튼 중 하나를 터치합니다:

```
[ 식음료 ] [ 교통 ] [ 사무용품 ] [ 숙박 ] [ 기타 ]
```

**OCR 신뢰도:**

화면에 OCR 정확도가 표시됩니다:
- 80% 이상 (🟢): 인식 결과가 정확할 가능성이 높음
- 80% 미만 (🟡): 직접 확인 후 수정 권장

> **중요**: 금액과 상호명이 실제 영수증과 다르면 **반드시 수정**하세요!

모든 내용을 확인한 후 **[확인 및 제출]** 버튼을 터치합니다.

### 14.4 Step 3: 검증 진행 (Processing 화면)

제출된 영수증이 자동으로 검증되는 과정을 실시간으로 보여줍니다.

**6단계 검증 과정:**

```
✅ 1. 이미지 업로드 완료
✅ 2. OCR 텍스트 추출 완료
✅ 3. 위치 검증 중...
⬜ 4. 업종 검증
⬜ 5. 한도 검증
⬜ 6. 최종 판정

진행률: ████░░░░░░  3/6
예상 소요 시간: 3초 이내
```

- 각 단계가 완료되면 ✅ 표시로 변합니다
- 보통 **3초 이내**에 모든 검증이 완료됩니다
- 완료되면 자동으로 결과 화면으로 이동합니다

> **30초 이상 걸리는 경우**: "처리 시간이 초과되었습니다" 메시지가 나타납니다. 거래 내역에서 나중에 확인할 수 있습니다.

## 17장. 설치 관련 오류

### 17.1 `npm install` 실패

**증상**: `npm install` 실행 시 빨간색 오류 메시지가 나타남

**오류 1: "node-gyp rebuild" 또는 "bcrypt" 관련 오류**

```
npm ERR! gyp ERR! build error
npm ERR! node-pre-gyp ERR! build error
```

**해결 방법:**

1. Windows Build Tools를 설치합니다:
   ```
   npm install --global windows-build-tools
   ```
2. 다시 시도합니다:
   ```
   npm install
   ```

**오류 2: "ERESOLVE unable to resolve dependency tree"**

```
npm ERR! ERESOLVE unable to resolve dependency tree
```

**해결 방법:**
```
npm install --legacy-peer-deps
```

**오류 3: 권한 오류 (EPERM)**

```
npm ERR! code EPERM
npm ERR! Error: EPERM: operation not permitted
```

**해결 방법:**

1. 명령 프롬프트를 **관리자 권한**으로 실행합니다
   - 시작 메뉴 → "cmd" 검색 → 마우스 오른쪽 클릭 → "관리자 권한으로 실행"
2. 다시 `npm install` 실행

---

### 17.2 Node.js 설치 오류

**증상**: `node --version` 입력 시 "인식할 수 없는 명령"

```
'node'은(는) 내부 또는 외부 명령, 실행할 수 있는 프로그램, 또는
배치 파일이 아닙니다.
```

**해결 방법:**

1. 컴퓨터를 **재시작**합니다
2. 재시작 후에도 안 되면, 환경 변수 PATH에 Node.js가 추가되었는지 확인합니다:
   - `Win + R` → `sysdm.cpl` → **고급** 탭 → **환경 변수**
   - "Path" 변수에 `C:\Program Files\nodejs\`가 있는지 확인
   - 없다면 **편집** → **새로 만들기** → `C:\Program Files\nodejs\` 추가
3. Node.js를 완전히 삭제하고 다시 설치합니다

---

### 17.3 Docker Desktop 설치 오류

**증상**: "WSL 2 installation is incomplete" 또는 "WSL update required"

**해결 방법:**

1. 명령 프롬프트를 관리자로 열고:
   ```
   wsl --install
   ```
2. 컴퓨터를 재시작합니다
3. Docker Desktop을 다시 실행합니다

**증상**: "Docker Desktop requires a newer WSL kernel version"

**해결 방법:**

1. 명령 프롬프트를 관리자로 열고:
   ```
   wsl --update
   ```
2. Docker Desktop을 재시작합니다

## 18장. 서버 실행 오류

### 18.1 Docker 컨테이너가 시작되지 않음

**증상**: `docker compose ps`에서 "Exited" 상태

```
NAME                STATUS
corporate-card-db   Exited (1)
```

**해결 방법:**

1. 로그를 확인합니다:
   ```
   docker compose logs postgres
   ```
2. 이전 데이터가 손상된 경우, 초기화합니다:
   ```
   docker compose down -v
   docker compose up -d
   ```
   > ⚠️ `-v` 옵션은 모든 데이터를 삭제합니다. 중요한 데이터가 있다면 백업 후 실행하세요.

3. 포트가 이미 사용 중인 경우:
   ```
   netstat -ano | findstr :5432
   ```
   다른 프로그램이 5432 포트를 사용하고 있다면, 해당 프로그램을 종료합니다.

---

### 18.2 API 서버가 시작되지 않음

**증상**: `npm run api:dev` 실행 시 오류

**오류 1: "Unable to connect to the database"**

```
[TypeOrmModule] Unable to connect to the database
Error: connect ECONNREFUSED 127.0.0.1:5432
```

**원인**: PostgreSQL이 실행되지 않음
**해결**: Docker 컨테이너 상태를 확인하고, 필요하면 다시 시작합니다:
```
docker compose up -d
```

**오류 2: "password authentication failed"**

```
error: password authentication failed for user "postgres"
```

**원인**: `.env.development`의 `DB_PASSWORD`가 Docker 설정과 불일치
**해결**: `.env.development` 파일을 열어 `DB_PASSWORD=password`로 수정합니다.

**오류 3: "Port 3000 is already in use"**

**원인**: 이전 API 서버 프로세스가 아직 실행 중
**해결**:
```
taskkill /F /PID $(netstat -ano | findstr :3000 | head -1 | awk '{print $NF}')
```
또는 작업 관리자(Ctrl+Shift+Esc)에서 `node.exe` 프로세스를 종료합니다.

---

### 18.3 대시보드가 표시되지 않음

**증상**: `http://localhost:5173` 접속 시 "사이트에 연결할 수 없음"

**확인 사항:**

1. 대시보드 서버가 실행 중인지 확인합니다
   - `npm run dashboard:dev`를 실행한 명령 프롬프트 창이 열려 있는지 확인
2. 포트를 확인합니다
   - 대시보드가 다른 포트(5174 등)에서 실행 중일 수 있습니다
   - 실행 로그에서 "Local: http://localhost:XXXX" 부분을 확인하세요
3. 방화벽이 차단하고 있는지 확인합니다
   - Windows 보안 → 방화벽 → Node.js 허용 확인

## 19장. 로그인 및 인증 오류

### 19.1 로그인 직후 바로 로그아웃됨

**증상**: 로그인은 성공하지만, 대시보드로 이동하면 바로 로그인 화면으로 돌아감

**원인**: JWT 토큰이 너무 빨리 만료됨 (3초)

**해결 방법:**

`packages/api/src/modules/auth/auth.service.ts` 파일에서 `JWT_ACCESS_EXPIRATION` 값을 `Number()`로 감싸야 합니다:

```typescript
// 수정 전 (❌ 잘못된 코드)
const expiresIn = this.configService.get<number>('JWT_ACCESS_EXPIRATION', 3600);

// 수정 후 (✅ 올바른 코드)
const expiresIn = Number(this.configService.get('JWT_ACCESS_EXPIRATION', '3600'));
```

> 이 버그는 v1.0.0에서 이미 수정되어 있습니다. 이전 버전을 사용 중이라면 업데이트하세요.

### 19.2 "계정이 잠겼습니다"

**증상**: "계정이 잠겼습니다. 30분 후 다시 시도하거나 관리자에게 문의하세요."

**원인**: 비밀번호를 5회 이상 잘못 입력

**해결 방법:**

1. **30분 대기** 후 재시도합니다
2. 또는 관리자가 Redis에서 잠금을 해제합니다:
   ```
   docker exec -it corporate-card-redis redis-cli DEL "login:locked:사번"
   ```
   예시: `DEL "login:locked:EMP001"`

### 19.3 "Network Error"

**증상**: 로그인 버튼 클릭 시 "Network Error" 또는 "ERR_CONNECTION_REFUSED"

**원인**: API 서버가 실행되지 않음

**해결 방법:**

1. API 서버 실행 확인:
   ```
   curl http://localhost:3000/api/v1/health
   ```
2. 응답이 없으면 API 서버를 시작합니다:
   ```
   npm run api:dev
   ```

## 20장. 대시보드 오류

### 20.1 데이터가 표시되지 않음

**증상**: 대시보드에 "데이터가 없습니다" 또는 빈 표시

**원인 1**: 시드 데이터가 로드되지 않음
**해결**: Docker 컨테이너를 초기화합니다:
```
docker compose down -v
docker compose up -d
```

**원인 2**: API 서버 연결 문제
**해결**: 브라우저 개발자 도구(F12) → Network 탭에서 빨간색 오류 요청을 확인합니다.

### 20.2 차트가 보이지 않음

**증상**: 통계 페이지에서 차트가 빈 영역으로 표시됨

**해결**:

1. 기간 필터를 확인합니다 (시드 데이터의 날짜 범위 내인지)
2. 브라우저 캐시를 삭제합니다: `Ctrl + Shift + Delete`
3. 브라우저를 새로고침합니다: `Ctrl + Shift + R` (하드 리로드)

### 20.3 설정 저장이 안 됨

**증상**: "저장" 버튼 클릭 시 오류

**확인 사항:**

1. 관리자(ADMIN001) 계정으로 로그인되어 있는지 확인
2. API 서버 콘솔에 오류 메시지가 있는지 확인
3. 필수 입력 항목을 모두 채웠는지 확인

## 21장. 모바일 앱 오류

### 21.1 카메라가 작동하지 않음

**증상**: 영수증 촬영 시 카메라가 열리지 않음

**해결 방법:**

1. 앱에 카메라 권한이 부여되었는지 확인합니다
   - (Android) 설정 → 앱 → 법인카드 앱 → 권한 → 카메라: 허용
   - (iOS) 설정 → 법인카드 앱 → 카메라: 허용
2. 다른 앱이 카메라를 사용 중이 아닌지 확인합니다
3. 앱을 종료했다가 다시 실행합니다

### 21.2 GPS 위치가 잡히지 않음

**증상**: "위치를 가져올 수 없습니다" 또는 정확도가 "나쁨"

**해결 방법:**

1. 위치(GPS) 권한이 허용되었는지 확인합니다
   - (Android) 설정 → 위치 → 켜기 & 앱 권한 확인
   - (iOS) 설정 → 개인 정보 보호 → 위치 서비스 → 앱: "앱 사용 중" 선택
2. 실외로 이동하여 GPS 수신을 개선합니다
3. Wi-Fi를 켜면 실내에서도 위치 정확도가 향상됩니다
4. **[위치 재시도]** 버튼을 터치합니다

### 21.3 영수증 업로드 실패

**증상**: "업로드에 실패했습니다" 오류

**확인 사항:**

| 확인 항목 | 해결 방법 |
|-----------|-----------|
| 인터넷 연결 | Wi-Fi 또는 데이터 연결 확인 |
| 파일 크기 | 10MB 이하인지 확인 (고화질로 찍으면 초과될 수 있음) |
| 서버 상태 | API 서버가 실행 중인지 확인 |
| 저장 공간 | 기기 저장 공간이 충분한지 확인 |

### 21.4 검증이 30초 이상 걸림

**증상**: Processing 화면에서 진행이 멈춤

**해결 방법:**

1. "거래 내역에서 나중에 확인" 안내에 따라 홈으로 이동합니다
2. 잠시 후 거래 내역 탭에서 해당 거래를 확인합니다
3. 반복 발생 시 API 서버 로그를 확인합니다:
   - OCR 서비스 연결 문제일 수 있습니다
   - 관리자에게 설정 → 외부 서비스 → OCR 연결 상태를 확인 요청합니다
